# MediaCrawler：把自媒体数据采集放进可控的学习边界

MediaCrawler 是一个面向学习和研究的多平台自媒体数据采集项目。它覆盖小红书、抖音、快手、B 站、微博、贴吧、知乎等常见平台，并把关键词搜索、指定帖子、二级评论、创作者主页、登录态缓存、代理池和评论词云图等能力放在同一个工具框架里。

这类项目最需要先讲清楚的不是“能不能抓”，而是“边界在哪里”。项目 README 和在线文档都反复强调，它只应被用于学习与技术研究，不能用于商业用途、侵权用途或大规模扰动平台的行为。把这条边界放在文章开头，是理解 MediaCrawler 的前提。

## 核心思路是浏览器自动化，而不是硬解所有签名

从 README 看，MediaCrawler 的主要技术路线是基于 Playwright 保存和复用浏览器登录态。它不是把每个平台的加密签名逻辑都拆成独立逆向工程，而是尽量利用带登录状态的浏览器上下文，通过页面环境获取请求所需的参数。

这个选择降低了入门门槛，也让项目更像一个“平台数据采集实验台”：用户先完成登录，再在受控配置下执行搜索、详情、评论或创作者主页等任务。对学习者来说，重点不只是看到数据落盘，而是理解浏览器自动化、登录态、请求参数、数据模型和存储层如何协作。

文档还提到 CDP 模式，也就是通过 Chrome DevTools Protocol 连接用户本机已有的 Chrome 或 Edge 浏览器。这个模式能复用更真实的浏览器环境和登录状态，但同时也提高了合规和隐私要求。任何使用登录态、Cookie 或浏览器数据的流程，都不应该被当成可公开发布的材料。

## 多平台能力依赖统一的工程骨架

项目的公开结构文档把系统拆成几类清楚的模块：抽象爬虫基类、浏览器数据目录、配置目录、平台实现、数据模型、工具函数、数据库入口和主程序入口。平台目录下再分别承载抖音、小红书、B 站、快手等平台的具体实现。

这种结构的价值在于把“每个平台不同”与“工具链共同部分”分开。共同部分负责浏览器、配置、存储、工具函数和上下文变量；平台部分则处理各自的数据入口、页面行为和字段模型。对一个多平台采集工具来说，这比把所有逻辑堆进一个入口脚本更容易维护。

依赖清单也反映了这个定位。Python 侧使用 Playwright、httpx、pydantic、parsel、pandas、aiosqlite、aiomysql、redis、jieba、wordcloud、matplotlib 等库，既覆盖浏览器自动化和 HTTP 请求，也覆盖数据校验、解析、存储和中文文本分析。Node.js 主要服务于部分平台依赖的 JavaScript 执行环境，文档要求版本不低于 16。

## 数据保存不是终点，数据治理才是重点

MediaCrawler 支持多种保存方式：SQLite、MySQL、CSV 和 JSON。README 建议个人小规模使用时优先选择 SQLite，因为它不需要额外数据库服务，适合本地实验。对于更复杂的场景，MySQL 可以承载关系型存储，但需要用户自行初始化数据库结构。

词云图功能则把评论数据进一步转成可视化结果。公开文档说明，词云目前依赖 JSON 保存模式，并需要开启评论采集和词云配置；用户还可以配置自定义词和停用词文件，中文字体文件也会影响最终图像输出。

这些功能容易让项目看起来像“采集越多越好”，但真正稳妥的用法恰恰相反：只采集必要的公开信息，控制范围和频率，不保存不需要的个人数据，不公开登录态、Cookie、数据库文件或原始大批量数据集。尤其是评论和用户主页相关内容，更应该以最小化和脱敏为默认原则。

## 运行体验依赖环境和平台状态

从使用文档和常见问题看，MediaCrawler 的运行体验会受到浏览器驱动、Node.js、登录验证、平台风控、网络环境和配置项影响。比如，某些平台需要 Node.js 才能执行相关逻辑；扫码登录后可能遇到滑块验证；运行一段时间后也可能因为账号触发平台风控而失效。

这说明它不是一个可以无脑运行的稳定采集服务，而是一个需要理解环境、配置和目标平台规则的研究工具。Playwright、CDP、登录态缓存、代理配置和词云分析都只是工程手段，不能替代对平台条款、法律法规和数据伦理的判断。

项目文档中的免责声明值得保留为核心设计约束：学习、研究、低强度、合规、尊重平台和用户权益。没有这几个前提，多平台采集工具的技术能力越强，风险也越大。

## 小结

MediaCrawler 的价值不在于鼓励大规模抓取，而在于提供一个观察多平台采集工程的样本：Playwright 登录态、可选 CDP 模式、多平台实现、结构化存储、词云分析和文档化配置共同组成了一个完整练习场。

本次整理只依据 README、package/pyproject/requirements 依赖清单，以及公开 docs 中的使用、CDP、项目结构、词云、原生环境和常见问题文档。没有读取源码全文、账号配置、浏览器数据、数据库、采集结果、图片二维码、字体文件、安装包或任何私密材料。
